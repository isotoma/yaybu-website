Yaybu a modern, pythonic configuration engine

What is it

A configuration management engine written in Python. You write some configuration:

resources.append:
  - Package:
      name: git-core
  - Checkout:
      name: /var/local/checkouts/readthedocs
      repository: http://github.com/rtfd/readthedocs.org.git
  - Execute:
      name: install-requirements
      command: pip install -r pip_requirements.txt
      cwd: /var/local/checkouts/readthedocs
      policy:
        execute:
          when: sync
          on: Checkout[/var/local/checkouts/readthedocs]

And you run it:

$ yaybu example.yay

Yay a shiny new configuration language

Yay is the flexible YAML based language Yaybu uses for defining server configurations in. But its not just for Yaybu.

Simple use of Yay gives you Python dictionaries of your config file (after resolving any variables, conditionals and loops)

import yay
config = yay.load_uri("/tmp/myfile.yay")
config["somedata"][0]

Sidekick the Software Development Kit Development Kit

Want a reproducible development environment with snapshotting? Done. Want to be able to setup a cloud instance to demo current progress to the client? Done. Want to be able to write tests against your infrastructure? Done.

Define your nodes in your Sidekick file and run sidekick up to get a working deployment.

nodes:
  - name: web
    base: http://localhost/bases/lucid-amd64.tar.gz
    yaybu:
      recipe: web.yay

Caroline your new boss

We love buildbot. I've lost count of how many i've snook into use at work. Caroline is one of them. She watches you and tells you off when you break the deployment.

You can configure Caroline to automatically maintain a running test instance of your work, rolling back to a working version if a deployment fails for any reason, with a few lines of Yay configuration. Caroline can also be configured to run any infrastructure level tests when your code changes.

A config might like something like this:

projects:
  - name: example
    repo:
      type: git
      url: git://github.com/Jc2k/sidekick-example

    on-failure: rollback
    on-success: leave-running
